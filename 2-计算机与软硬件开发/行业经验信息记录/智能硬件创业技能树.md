> **电子硬件，软件系统，工业设计，供应链，可靠性与合规测试，定价与利润，营销和渠道，融资和管理**

<a name="U7xPR"></a>
#### 1、电子硬件：电路原理图->电路板
原理图文件叫SCH，线路板就是PCB，设计工具常见有Altium Designer（Protel），高端的有PADS，Allegro，免费开源的有KiCAD，立创EDA，工具之间并不兼容。<br />芯片需由芯片原厂提供数据手册（Datasheet），也叫规格书，了解芯片性能参数和封装。<br />量产文件是发给工厂生产的。一般包括线路板制版Gerber文件，还有SMT贴片需要的BOM表（物料清单）、钢网、坐标等文件。工厂根据Gerber印制线路板，贴片厂则按照BOM文件采购物料，然后根据坐标文件来贴片。钢网的作用是用来给线路板上锡膏用的，而坐标文件则告诉贴片机，每个元器件的坐标位置。国内现在也有很多配套的工厂，除了生产PCB外，一些简单的贴片（SMT）也可以完成。整个流程结束之后，出品就叫PCBA，也就是把电子元器件和芯片都焊接在PCB电路板上。

<a name="D8Shk"></a>
#### 2、软件系统：任何嵌入式硬件都需要一个操作系统来运行  
通过编译工具，将源代码编译成目标文件，编译工具常见的有Keil，IAR等，近几年VSCODE，Eclipse也很火。嵌入式支持的编程语言，C语言比较常见，估计有90%吧，C++大约在linux以上的系统才会用到，一些低端的芯片也还支持汇编语言，Python使用较少。<br />通过编译器生成的目标文件，常见的有Bin和Hex格式，Bin文件就是十六进制代码，而Hex是Intel公司的格式，可以理解为十六进制代码的可阅读化。目标文件是芯片能识别的机器语言，比汇编语言更低级，也就是我们常说的固件（firmware），编译结束后通过烧录器将固件烧录到芯片上，开机就可以运行了。<br />一般小型的MCU，可以跑个RTOS系统，或者裸机运行代码，相对应的CPU很多，一般8位的有51和AVR指令集，32位基本上都是ARM M内核，典型芯片的是STM32。这里不得不提一下，这两年国产的嵌入式系统和芯片发展很快，ESP32靠Wifi SOC独步天下，软件方面则有RTThread、鸿蒙（LiteOS）这些系统和开发工具，基本上全国产化没太大问题。<br />项目如果复杂程度略高，比如需要一个大的显示屏，需要网络交互或者数据处理，就需要一个中型一点的系统，可选方案是Linux系统，但Linux嵌入式系统开发不算太友好，相对来说开发也比较耗时间，另外支持Linux系统的芯片成本也略高，需要有MMU（内存管理单元），一般是ARM A内核以上；再大型就是android了，跑android系统的硬件，开发容易，也可以兼容第三方的APP，缺点就是成本比较高。<br />软件开发过程中，需要调试时有几种方式，一种是软件调试，keil支持纯代码单步调试，另一种是通过仿真器，再结合编译工具。但如果这些都失效，还可以使用逻辑分析仪，在很多时序电路上，逻辑分析仪是一个神器，可以帮助排查最底层的问题。<br />除了软件和系统之外，通信也是电子电路很重要的一个概念，比如无线通信有wifi，蓝牙，4G，zigbee，lora等等，无线通信在速率，距离，功耗等各有不同，需要看具体的场景来评估。而有线通信有串口，SPI，USB，HDMI，RJ45网口等，电路内的协议显示屏或摄像头用的MIPI，DVP等接口。

#### 3、工业设计：包括外观设计（ID）和结构设计（MD）
外观除了产品的整体形态设计之外，还包括了CMF（颜色、材料、表面处理），而结构设计也非常重要，关乎产品的品质、成本和可靠性，必须熟悉知道每种材料的特性，比如PC、ABS或者硅胶、TPU等等。一般结构设计之后到模具厂，还需要进行DFM（设计为了可量产），也称之为可靠性评审。<br />结构设计结束之后，一般会找专业公司做外观手办和结构手办，如果是早期验证阶段，也可以通过3D打印的方式，国内已经有好多个平台可以直接上传stp或stl文件，直接在线下单即可。

#### 4、供应链和制造：成本控制和品质控制
他决定产品的成本、品质、稳定性。一般来说，供应链问题涉及到电子元器件物料的采购，模具厂、贴片厂、组装厂等外部工厂的配合。<br />如果是找外协的加工厂代工生产，必须要有一个专门的驻厂人员负责品质控制（QC），研发人员必须出具非常详细的作业指导书，关键测试点上必须有自己制作的测试夹具，然后傻瓜化操作，把问题控制在流水线上。<br />如果是非常注重产品的外观，除了设计的工作要做足之外，模具的成本万万不可节省。<br />关于品质控制，有六西格玛（6sigma）次品率标准，具体数据是：6sigma=3.4/100万; 5sigma=2.3/1万；4sigma=6.2/1000；3sigma=6.6/100。<br />项目流程：Planning -> EVT工程验证测试 -> DVT设计验证测试 -> PVT生产验证测试 -> PP小批量生产 -> MP量产阶段

<a name="NlPDd"></a>
#### 5、可靠性测试和合规认证
主要是看各部件的质量和使用寿命，另外在一些极端环境下的使用情况，在电子元器件领域，一般有几个标准，比如消费级，工业级，军工级和车规级。<br />几种常见的可靠性测试：<br />1）气候环境：高低温存储，高低温工作，防水防尘实验；2）机械环境：裸机震动，包装震动，自由跌落，包装跌落；3）使用寿命：按键寿命，拔插寿命；触摸屏寿命；滚筒跌落；4）生物化学：汗液酒精油污；抗赃污；耐磨实验<br />一般来说，中国是CCC认证，美国是FCC，欧盟是CE和环保ROHS。这些都是强制性的认证。除了CCC这种强制类认证之外，还有各品类的执行标准，比如电子产品的执行标准是GB4943，家用电器是GB4706，如果包含了WIFI和蓝牙，还需要SRRC的无线电认证，如果是3G/4G产品，还需要CTA入网许可等。（特里：因为玩具类的产品要求在充电的时候，产品不可以使用）

<a name="SLX5n"></a>
#### 6、产品定价与利润分配
计算公式：出厂价= 成本价 / (1- 毛利率)。如果产品小于20%的毛利率，可能都很难生存了。把整体的成本加上20-30%的毛利率空间，就是产品的出厂价，正常来讲不会直接到消费者手上，还需要经过一些渠道，京东天猫也是渠道之一，这些渠道都是有成本的，而渠道的链条越长，成本就越高。任何一个产品，都会从两个方面来核算成本，一个是从零售价倒算回来，一个是从成本价算回去，如果没法达到理想的价格，就必须调整方案了。从商业的角度，可以忍受亏损一定是看中长期的价值。

#### 7、市场营销和渠道销售
一般来说，市场部门是在花钱的，销售部门是在赚钱的。对于产品来讲，市场是收集用户需求和发现产品痛点的部门，而销售是反馈产品意见和改良产品的意见。

#### 8、融资和组织管理
理想的情况下，老大的股份要占有绝对优势，也就是要有一锤定音的权力。一般来说有几个点位可以注意一下，比如67%是绝对控股权，51%是相对控股权，34%是一票否决权。在合伙人的角色里，最好技术产品、市场营销、供应链管理，和CEO都有合适的人选，因为每一个环节都很重要，这是团队的基本盘。<br />每一轮的融资，都会同步稀释原始股份，当老大的股份小于51%时，可以和合伙人签署一致行动人协议书，以保证在董事会有更大话语权。融资的时机要把握好，我现在觉得，能不融资尽量不融资，融来的钱未来都是要加倍还回去的，投资机构都是锦上添花，从不雪中送炭，所以在财务数据和运营指标很不理想的情况下，不要浪费太多的时间在接触投资人上面，当然要节省时间，也可以聘请专业的FA（财务代理），他们在行业内有更多的资源，知道哪些投资公司的偏好，也许成功几率会更高些。
